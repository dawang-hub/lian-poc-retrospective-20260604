## 一、复盘目标

本次复盘围绕利安人寿APP改版POC项目的完整过程展开，目标是沉淀项目中的流程亮点、协作问题、角色能力短板和可复用方法，形成后续POC项目可执行的流程机制与行动项。

本次复盘重点不只评价最终产出，更关注以下问题：

1. POC项目启动、策略、交互、视觉、PPT包装各环节是否顺畅。
2. 团队首次磨合POC过程中，哪些协作方式有效，哪些造成了反复和焦虑。
3. 交互与视觉如何更早参与策略共创，减少后期理解断层。
4. 后续如何以机制化方式提升POC效率、表达质量和团队协同。

## 二、总体结论

本次利安人寿APP改版POC最终产出质量整体达标，团队对客户需求、业务背景和方案方向的理解较准确，最终方案在策略、视觉呈现和PPT表达上具备一定创新性和完整性。但项目过程消耗较高，团队在前期策略共创、交互视觉主线统一、PPT表达框架和POC节奏控制上存在明显磨合问题。

综合来看，本项目属于“结果可接受、过程代价偏高”的一次POC实践。核心问题不是单个成员能力不足，而是团队尚未形成稳定的POC工作节奏：前期没有充分完成策略、交互、视觉表达主线的共同定义，导致后续在页面细节、视觉风格、PPT逻辑上多次反复。

后续需要将“先定主线，再做页面；先共创表达，再分工执行”固化为POC项目机制。

## 四、做得好的地方

### 1. 启动会和定期对齐有效

项目开始阶段组织启动会，对项目目标、背景、分工和节点进行了基础对齐。项目过程中也通过定期碰头同步思路和解决方案，这对团队理解项目背景、维持方向一致起到了积极作用。

后续应保留启动会机制，并进一步标准化启动会内容。

### 2. 对客户需求和业务背景理解较准确

交互侧认为，团队对业务范围、用户需求和整体方案方向的理解较准确，项目过程中大方向没有发生颠覆性调整。这说明团队对客户资料和业务目标的判断基本有效，具备较好的需求吸收能力。

### 3. 最终方案质量经过打磨后明显提升

视觉方案前期存在中规中矩、缺少主线的问题，但在多轮指导和调整后，整体质感和高级感有所提升。团队也在过程中逐步明确优秀设计需要克制、聚焦、围绕核心概念展开，而不是依赖颜色或元素堆叠。

### 4. PPT规范提前确定后提升了收尾效率

后期PPT包装阶段提前明确了部分页面规范和整体风格，以平面化排版为主，减少了不必要的质感追求。相比以往多人长时间包装PPT，本次后期收尾效率有所提升。

### 5. 形成了部分可复用经验

本项目沉淀出若干可复用方向，包括：

1. POC启动会机制。
2. POC节奏检查表。
3. 交互方案分析产出流程：现状分析、用户旅程、问题总结、策略规划、方案设计。
4. AI生图稳定方法：如何产出可控、高质量、可用于设计表达的图片。
5. 视觉表达前置方法：先梳理视觉主线、关键词和设计内核，再进入页面设计。

## 五、主要问题与原因分析

### 1. 策略共创不足，导致交互和视觉理解断层

复盘中多位成员提到，策略部分主要由负责人输出，交互和视觉更多是在既有框架中执行，前期共创不够充分。这样导致成员知道“要做什么”，但不完全理解“为什么这样做”以及“如何把理念转化为交互和视觉表达”。

典型表现包括：

1. 视觉侧理解“共生共长”等理念，但没有进一步拆解其视觉路线。
2. 交互侧方案方向稳定，但在PPT表达和客户叙事上缺少统一框架。
3. 后期策略、交互、视觉在PPT中重新整合，造成反复修改。

根因：策略阶段没有同步完成交互策略、视觉策略、表达关键词和参考方向的共创。

### 2. 工作节奏偏向先铺页面，缺少先定主线

复盘中明确提到，视觉和交互前期节奏存在问题：过早进入页面和细节，没有先把大的主线、故事、关键词、视觉路线和交互表达方式理清。

POC阶段的重点应是建立客户记忆点和方案信任感，而不是过早陷入详细设计细节。团队在时间紧张时容易焦虑，进而更快进入细节执行，反而降低效率。

根因：团队对POC阶段、概念设计阶段、详细设计阶段的产出重点认知还不统一。

### 3. PPT表达框架反复，非专业客户视角不足

交互和视觉在表达设计理念时，整体PPT框架和逻辑不够清晰，经历了多轮调整。复盘中提到，客户并不会严格区分交互和视觉，他们需要看到的是一个完整、可理解、能解决业务问题的方案。

后续PPT表达应更多围绕：

1. 客户/用户痛点是什么。
2. 我们用什么设计理念解决。
3. 对业务和用户分别有什么价值。
4. 每个设计细节如何支撑这个价值。

根因：最终PPT框架和逻辑没有在策略阶段共同规划，而是后期各自内容汇总后再整合。

### 4. 交互表达缺少惊喜感，视觉表达受限

交互侧在业务逻辑和产品层面思考较充分，但在表现形式、布局样式、组合方式和前瞻性表达上不足。视觉侧也反馈，部分交互框架较普通，会限制后续视觉表达。

根因：交互产出更多关注“这里是否需要放什么业务内容”，较少关注“这个内容如何以更有吸引力、更有记忆点的方式呈现”。

### 5. 视觉侧前期缺少项目灵魂和表达主线

视觉侧前期更多处于执行和配合状态，没有充分阅读客户资料、品牌文化和业务理念，也没有把策略概念拆解成可执行的视觉关键词、情绪、材质、参考和路线。

根因：视觉设计师没有在策略阶段把自己放在“共同策划者”的角色中，导致后续设计容易依赖参考和试错，而不是围绕稳定主线推进。

### 6. 负责人一次性输出过多，团队吸收困难

负责人在复盘中提到，由于首次带团队做POC，对成员POC能力和反馈偏好了解不足，前期一次性输出较多，导致成员吸收困难，也加剧了焦虑和反复。

根因：缺少POC前的团队能力摸底机制和节奏检查机制。

## 六、角色复盘

### 冯梦莹：主策略及POC总负责人

做得好的地方：

1. 对需求理解和快速响应能力强，能快速吃透客户需求和标书背景。
2. 对POC阶段重点展示内容有较强判断力。
3. 能根据反馈快速调整方案方向和PPT表达，闭环能力较好。

未达预期：

1. 对整体POC节奏把控和交互、视觉设计师引导不足。
2. 视觉建议因不够自信，部分反馈不够果断，导致问题延后暴露。
3. 对团队POC能力了解不充分，前期输出节奏和信息量没有很好适配成员。

后续改进：

1. 控制一次性输出量，采用阶段化任务分配。
2. 对视觉反馈采用“2条总体感受 + 1条必须改问题 + 1条可选建议”的格式。
3. POC启动前组织团队回答“擅长、踩坑、反馈偏好”三问，形成协作偏好表。

### 杜妍：视觉设计

做得好的地方：

1. 能听取建议并快速调整视觉方案。
2. 在打磨过程中掌握了更克制、更高级的页面处理方法。
3. 找到了一套AI生图稳定方法，可用于后续沉淀。

未达预期：

1. 前期过早进入页面执行，设计效率较低。
2. 对启动页和AI动效理解不够深入，试错时间较多。
3. 没有充分阅读客户资料并提炼视觉主线，导致策略和设计出现断层。

后续改进：

1. 前期先阅读客户资料、品牌文化和项目理念。
2. 在视觉执行前先梳理关键词、情绪、视觉路线和核心故事。
3. 将自己从“执行者”切换为“共同策划者”，主动参与策略共创。
4. 输出AI生图方法文档，沉淀为团队资产。

### 房昱锟：交互设计

做得好的地方：

1. 对业务范围和用户需求理解准确。
2. 整体交互方案方向稳定，过程中没有发生大方向偏移。
3. 沉淀了POC交互方案分析流程。

未达预期：

1. 交互框架和表现形式较普通，缺少POC项目需要的惊喜感。
2. 内容表现和设计亮点包装不足，对视觉表达造成一定限制。
3. PPT表达和案例包装耗时较多，策略框架总结不够清晰。

后续改进：

1. 加强对前瞻产品、同类产品、创新交互样式的观察和沉淀。
2. 在输出交互方案后增加“惊喜感自检”。
3. 从客户能理解的语言出发表达方案：先讲用户痛点和业务价值，再拆解设计细节。
4. 在业务逻辑之外，更多思考内容的表现形式和包装方式。

## 七、流程沉淀

### 1. POC启动会必须保留并标准化

启动会应对齐以下内容：

1. 项目背景和客户目标。
2. 核心业务痛点。
3. POC目标和展示重点。
4. 项目时间、节点和预算工时。
5. 角色分工和协作方式。
6. 当前项目属于POC、概念设计还是详细设计，明确阶段产出重点。

### 2. 策略共创应前置到项目第一天

建议在项目启动后安排半天至一天进行策略共创，由负责人先给出初步理念和方向，交互、视觉共同阅读客户资料并参与拆解。

共创产出应包括：

1. 项目核心理念。
2. 用户与业务痛点。
3. 交互策略。
4. 视觉策略。
5. 关键词和表达情绪。
6. 参考方向和禁区。
7. PPT叙事主线。

### 3. 交互视觉必须同步定义表达主线

交互和视觉不应等策略结束后各自寻找方向，而应在策略阶段同步确定：

1. 交互如何体现策略。
2. 视觉如何体现策略。
3. 哪些页面承担记忆点。
4. 哪些设计细节服务业务价值。
5. 哪些表现形式能带来客户惊喜感。

### 4. PPT框架需要共同完成

最终PPT不是交互稿和视觉稿的简单汇总，而是面向客户的完整方案叙事。建议在PPT进入正式制作前，由策略、交互、视觉共同完成一版“叙事骨架”。

建议PPT表达结构：

1. 客户现状与用户痛点。
2. 方案核心理念。
3. 体验路径和关键场景。
4. 核心设计亮点。
5. 对用户的价值。
6. 对业务的价值。
7. 可落地与可延展方向。

### 5. 建立POC节奏检查表

每次进入下一阶段前，团队先确认当前处于哪个阶段，避免在策略阶段做详细设计，或在详细设计前才补策略。

建议检查节点：

1. 启动前：背景、目标、工时、分工是否对齐。
2. 策略前：客户资料是否已阅读。
3. 设计前：理念、关键词、交互策略、视觉策略是否已确认。
4. 产出中：是否每1至2天同步方向，是否存在跑偏。
5. PPT前：叙事主线和页面框架是否共同确认。
6. 交付前：是否完成质量、创新性、完整性和合理性自检。

## 九、后续建议

1. 将本次复盘行动项纳入团队下一个POC项目的启动流程，而不是作为单次总结停留。
2. 在下一次POC开始前，用1小时进行团队能力轻摸底，观察设计师产出速度、表达偏好和反馈吸收方式。
3. 建立“策略、交互、视觉、PPT表达”四张模板，减少每次从零开始摸索。
4. 将焦虑转化为节奏管理：时间越紧，越需要先用半天到一天理清主线，再进入页面输出。
5. 将“客户能记住什么”作为POC质量判断标准之一，避免只关注细节完整度。

## 十、关键共识

本次复盘最终形成的关键共识是：POC项目不是详细设计项目，核心目标是用有限时间建立客户信任、展示方案想法和形成记忆点。因此，团队后续需要优先提升策略共创、表达主线、交互视觉协同和PPT叙事能力。

下一次POC应遵循：先共创策略，再定义交互和视觉路线；先搭建客户能理解的故事，再进入页面细节；先统一目标和节奏，再分工执行。
